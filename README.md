# Курсовой проект

## Тема

Применение нейросетей для статистической обработки информации.

## Идея проекта

В проекте сравниваются несколько подходов к прогнозированию средней суточной температуры воздуха в Волгограде по историческим данным Meteostat:

- наивный прогноз;
- простой baseline на основе скользящего среднего;
- экспоненциальное сглаживание;
- классическая модель машинного обучения для табличных признаков;
- нейросетевая модель.

Проект сделан как учебно-исследовательский: сначала строится воспроизводимый датасет на уровне дней, затем на его основе выполняется моделирование и сравнение подходов.

## Подготовка окружения

### 1. Создание виртуального окружения

```bash
python -m venv .venv
```

### 2. Активация виртуального окружения

Windows PowerShell:

```powershell
.venv\Scripts\Activate.ps1
```

Windows cmd:

```cmd
.venv\Scripts\activate.bat
```

Linux / macOS:

```bash
source .venv/bin/activate
```

### 3. Установка зависимостей

Рекомендуется сначала обновить `pip`, а затем установить зависимости:

```bash
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

## Регистрация kernel для Jupyter

```bash
python -m ipykernel install --user --name kursovaya-neural-stats --display-name "Python (.venv) kursovaya"
```

После этого в Jupyter можно выбрать kernel `Python (.venv) kursovaya`.

## Как открыть ноутбуки

Из корня проекта выполните:

```bash
jupyter notebook
```

После запуска откройте папку `notebooks` и выберите нужный файл.

## Порядок запуска проекта

1. Запустите ноутбук `notebooks/01_dataset_creation.ipynb`.
2. Убедитесь, что создан итоговый CSV-файл датасета.
3. Затем запустите ноутбук `notebooks/02_modeling_and_comparison.ipynb`.

## Что создаётся в результате

После выполнения первого ноутбука будут созданы:

- `data/raw/volgograd_hourly_raw.csv` — сырые почасовые данные Meteostat;
- `data/raw/volgograd_daily_raw.csv` — сырые суточные данные Meteostat;
- `data/processed/volgograd_daily_temperature_dataset.csv` — итоговый подготовленный датасет для моделирования.

После выполнения второго ноутбука будут созданы:

- графики в папке `outputs/figures`;
- таблицы с метриками и анализом ошибок в папке `outputs/tables`.

## Краткое описание ноутбуков

### `01_dataset_creation.ipynb`

Ноутбук:

- выбирает подходящую станцию Meteostat рядом с Волгоградом;
- загружает почасовые и суточные данные;
- агрегирует почасовые данные до уровня дней;
- формирует календарные, лаговые и скользящие признаки;
- аккуратно обрабатывает пропуски;
- сохраняет итоговый датасет.

### `02_modeling_and_comparison.ipynb`

Ноутбук:

- загружает подготовленный датасет;
- формирует временные разбиения `train / validation / test`;
- строит baseline, статистическую модель, классическую ML-модель и нейросеть;
- сравнивает модели по `MAE` и `RMSE`;
- сохраняет итоговые таблицы и графики;
- делает вывод о том, дала ли нейросеть преимущество в данной постановке.

## Важные замечания

- Для загрузки данных через Meteostat в момент выполнения первого ноутбука нужен доступ в интернет.
- Проект не использует Google Colab и рассчитан на локальный запуск.
- Нейросетевая модель подобрана так, чтобы работать на обычном компьютере без GPU.
- В ноутбуке моделирования признаки подбираются так, чтобы не использовать информацию из будущего.
