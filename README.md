# Курсовой проект

## Тема

Применение нейросетей для статистической обработки информации.

## Идея проекта

Проект посвящён прогнозированию средней суточной температуры воздуха в Волгограде по историческим данным Meteostat. Работа построена как учебно-исследовательский пайплайн:

1. из почасовых и суточных данных Meteostat формируется воспроизводимый дневной датасет;
2. затем сравниваются статистические, классические ML- и нейросетевые модели;
3. отдельно исследуется, как меняется качество моделей при разной длине обучающей выборки при фиксированных validation и test интервалах.

## Что сравнивается

В ноутбуке моделирования рассматриваются:

- `Naive (t-1)` — прогноз по значению предыдущего дня;
- `Seasonal naive (t-365)` — прогноз по значению примерно того же дня прошлого года;
- `Seasonal AutoReg` из `statsmodels` как сезонный статистический baseline;
- `RandomForestRegressor` как сильная и при этом понятная табличная ML-модель;
- `Dense`-сеть по лаговым и календарным признакам;
- `GRU` по последовательностным окнам длиной 30 дней;
- `Conv1D` по тем же временным окнам.

Основные метрики:

- `MAE`
- `RMSE`
- `MASE`
- `bias`

## Воспроизводимость

В проекте зафиксированы:

- конечная дата используемого датасета: `2025-12-31`;
- validation-интервал: `2024-01-01 ... 2024-12-31`;
- test-интервал: `2025-01-01 ... 2025-12-31`;
- random seed для моделей и нейросетей.

Все эти настройки собраны в `src/constants.py`.

## Подготовка окружения

### 1. Создание виртуального окружения

Linux/macOS/WSL:


Windows:

```bash
python -m venv .venv
```

Windows PowerShell:

```powershell

```

Windows cmd:


```bash
source .venv/bin/activate
```

```bash

```

## Порядок запуска

1. Запустите `notebooks/01_dataset_creation.ipynb`.
2. Убедитесь, что создан файл `data/processed/volgograd_daily_temperature_dataset.csv`.
3. Затем запустите `notebooks/02_modeling_and_comparison.ipynb`.

Важно: второй ноутбук опирается на результаты первого.

## Что делает первый ноутбук

`notebooks/01_dataset_creation.ipynb`:

- загружает данные Meteostat для Волгограда;
- пытается выбрать ближайшую подходящую станцию;
- при сетевой ошибке может использовать сохранённые raw-файлы из `data/raw`;
- агрегирует hourly-данные до уровня дней;
- объединяет hourly-признаки и fallback из `Daily`;
- создаёт календарные, лаговые и скользящие признаки;
- аккуратно обрабатывает пропуски без использования будущих данных;
- сохраняет итоговый датасет в `data/processed/volgograd_daily_temperature_dataset.csv`.

## Что делает второй ноутбук

`notebooks/02_modeling_and_comparison.ipynb`:

- загружает подготовленный датасет и ограничивает его фиксированным периодом;
- в начале автоматически удаляет устаревшие `02_*` артефакты предыдущих версий из `outputs/figures` и `outputs/tables`;
- проверяет покрытие по годам и формирует сценарии длины train;
- сравнивает модели при одинаковых validation и test интервалах;
- считает `MAE`, `RMSE`, `MASE` и `bias`;
- строит итоговые таблицы на validation и test;
- делает отдельный анализ качества при разной длине train;
- строит графики прогнозов, кривых обучения и сезонных ошибок;
- формулирует исследовательские выводы о том, когда нейросети выигрывают, а когда нет.

## Какие файлы создаются локально

Во время выполнения проекта локально создаются или обновляются:

- `data/raw/volgograd_hourly_raw.csv`
- `data/raw/volgograd_daily_raw.csv`
- `data/raw/meteostat_cache/`
- `data/processed/volgograd_daily_temperature_dataset.csv`
- таблицы в `outputs/tables/`
- графики в `outputs/figures/`

## Что лежит в репозитории, а что генерируется локально

Обычно в репозиторий коммитятся:

- исходный код из `src/`;
- ноутбуки из `notebooks/`;
- `README.md`, `requirements.txt`, `.gitignore`.

Локально генерируемые артефакты по умолчанию **не коммитятся**:

- raw-выгрузки из `data/raw/*.csv`;
- обработанный датасет из `data/processed/*.csv`;
- кэш Meteostat;
- графики и таблицы из `outputs/`.

Это поведение задано в `.gitignore`. При необходимости для сдачи курсовой вы можете отдельно добавить нужные итоговые CSV или изображения вручную.

## Важные замечания

- Для первой загрузки данных через Meteostat нужен доступ в интернет.
- Проект рассчитан на локальный запуск и не использует Google Colab.
- Нейросетевые модели сделаны умеренными по размеру и времени обучения, чтобы запускаться без GPU.
- Во втором ноутбуке не используется случайный `shuffle`, чтобы избежать утечки информации из будущего.
- Если нейросеть не становится лучшей моделью, это считается нормальным исследовательским результатом, а не ошибкой проекта.
